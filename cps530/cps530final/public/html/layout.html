In this tutorial, we are going to make a beautiful landing page. A landing page is page where businesses can present who they are, what they do, what they offer, etc. Businesses want to have a landing page in order for them to have an online presence, so their customers, clients or business partners can quickly see what they are up to.

Since landing page is an important part of business in the 21st century. The landing page itself needs to look appealing, as it is a representation of the business itself on the web. But, what makes a website appealing? a website is appealing as long as it caters to user needs and maximize user experience, it also needs to have structure.

In terms of landing page, we can divide the sections into 4. We need navigation bar for users to easily navigate on our website. Landing page also usually contains a hero, the hero can be used to present the business product or services, usually in the hero section, there is also a call-to-action button. we also need the contents of the business and lastly the footer. in this tutorial, we are going to build these sections from top to bottom using Bulma.


Set Up environment

Before we start building on Bulma, we need to set up our environment, we can grab a starter template on Bulma or we can get Bulma's CSS file from the provided CDN.


What we do here is we just attach a minified CSS provided by Bulma, it will automatically fetch the CSS from the CDN so we don't need to have file locally in order to work with Bulma.

Setting up the Hero section

We are going to start building page from top to bottom. Inside of the body tag, we want to set up of Hero section. In Bulma, Hero is build from section and it has 2 sub-sections. We can specify these two sections by referring them with their class hero-head and hero-body

In the parent section tag, we can specify the background to be used in the Hero and adjust the background property as needed.

Since we want to have our navigation bar on top of the hero section, we can write the navbar as part of hero-head, this navbar also have to be mobile responsive so that users still have the best experience when browsing the website.

In the hero-body, we can have the image, title of the website, and call-to-action button. In essence, we put every content we want to show in the Hero in hero-body.


Building the Contents

Next, we want to setup the content of the website itself. The idea of building the content in Bulma is easy as it is similar to other frameworks like Bootstrap.

Before we start building the content, we need to understand the concept of column in Bulma. To build a column we need to have a columns container. This container will hold all the sub-column. The column itself have equal width within them, for example if you have 2 columns, the container will be divided by 50:50 for each column, it goes the same way if you have 4 columns, the container will be divided by 4 and each column will fill out 25% of the width of columns container.

In this section, we will have a columns container that will have 2 subcolumn, one column to hold image and another column is to hold the description. We also put the columns in a container so that it is always in the center of the page.

Since we understand the concept of column in bulma, we can stretch its ability beyond dividing the sub columns equally. Here, what we do is we just add another class property to our div tag, we specify that these are boxes so it has border-radius property and make the corners look rounded by using box property. We also specify the size like is-one-quarter or is-full wide. These are similar approach that Pinterest used to make their home feed.

In the last part of the content page we build a medium hero section, the concept is the same as in building our first Hero, the difference is we specify the size of this Hero as is-medium. We also close our body tag by the end Hero section.


Set up Footers

After you close the body tag, we setup the footer by specify the footer tag along with the content you want to have in the footer. In this part, we put every content of the footer in the container. Then we override, the img to the left by specifying is-pulled-left while the rest of the content is in the center.
